### [tRPC](https://trpc.io/) and [Prisma](https://www.prisma.io/)

Even with TypeScript, I still face some problems regarding type safety in my projects. Next's APIs routes are amazing, but at the moment we still have to define an Interface or Type to help with intellisense while consuming the endpoints we created (or just live in the dark like js ğŸ¤·â€â™‚ï¸).

With supabase I have a similar problem: I can create databases really quickly but I still have to create interfaces or types myself.

It's not hard to do so, but the problem is that I now have 2 sources of truth in both cases. So if I make a change to the API response or the database tables/fields I'll have to update their types manually ğŸ˜¡. Again, that's not hard but it definitely opens up space for mistakes regarding type safety.

Prisma will allow me to create tables that are tied to the types I declared in my code. If I update that type, the database will follow and its queries will be typed without more effort.

With tRPC it's possible to _infer_ API response's types instead of _declaring_ them. Any changes to the endpoint's response won't need changing its interface because it its **inferred** and that's AWESOME.
